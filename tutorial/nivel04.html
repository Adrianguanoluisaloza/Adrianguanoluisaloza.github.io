🧭 ¿Qué es la etiqueta NAV?

La etiqueta <nav> es un contenedor semántico especial para menús de navegación. Introducida en HTML5, identifica las secciones de navegación principales de tu sitio web, ayudando a los buscadores y lectores de pantalla a entender la estructura de tu página.

📍 Ubicación típica: Se coloca dentro del <header> o directamente en el <body>, dependiendo del diseño.

💡 Analogías para entender NAV

1. Como un mapa de centro comercial:

  • 🏬 El <nav> = Los directorios con "Planta 1: Ropa, Planta 2: Comida"
  • 🧭 Te ayuda a encontrar rápidamente lo que buscas
  • 📍 Siempre está en lugares estratégicos (entrada, ascensores)

2. Como el índice de un libro:

  • 📖 Capítulo 1 → página 10
  • 📖 Capítulo 2 → página 45
  • 📖 Te permite saltar directamente a la sección que necesitas

3. Como las señales de tráfico:

  • 🚦 "Madrid ← 50km", "Barcelona → 100km"
  • 🛣️ Guían tu viaje indicando rutas y destinos
  • ⚡ Sin ellas, estarías perdido

🎯 ¿Por qué usar NAV en lugar de DIV?

Podrías usar un <div class="menu"> para tu menú, pero <nav> tiene ventajas importantes:

  • 🔍 SEO: Google identifica tu navegación principal y la indexa mejor
  • ♿ Accesibilidad: Lectores de pantalla anuncian "navegación" y permiten saltarla
  • 🧹 Código limpio: Otros desarrolladores entienden inmediatamente qué es
  • 📱 Navegadores: Pueden aplicar estilos especiales para móviles

✅ Correcto: <nav><ul><li>...</nav>

❌ Incorrecto (anticuado): <div class="menu">...</div>

📋 Estructura típica de un menú:

Un menú normalmente tiene esta jerarquía:

  • <nav> - Contenedor semántico principal
  • <ul> - Lista no ordenada (Unordered List)
  • <li> - Elementos individuales de la lista (List Item)
  • <a> - Enlaces (Anchors) dentro de cada li
<nav> ← Contenedor semántico
  <ul> ← Lista de opciones
    <li> ← Primer elemento
      <a href="/">Inicio</a>
    </li>
    <li> ← Segundo elemento
      <a href="/about">Acerca</a>
    </li>
    <li> ← Tercer elemento
      <a href="/contact">Contacto</a>
    </li>
  </ul>
</nav>

🤔 ¿Por qué usamos listas (<ul> y <li>)?

Porque un menú ES una lista de opciones. Usar <ul> + <li> es semánticamente correcto. Los lectores de pantalla pueden anunciar "lista de 5 elementos" y los usuarios pueden navegar entre ellos.

Todo sobre el atributo HREF

El atributo href (Hypertext REFerence) es el CORAZÓN de la navegación web. Define el destino del enlace. Tiene varios tipos:

🎯 Tipos de HREF

1. 📄 Enlaces relativos (mismo sitio)

<a href="contacto.html">Contacto</a> ← Archivo en la misma carpeta
<a href="paginas/about.html">Acerca</a> ← Archivo en subcarpeta
<a href="../index.html">Volver</a> ← Subir un nivel (..) y abrir index
<a href="/inicio.html">Home</a> ← Desde la raíz del sitio

✅ Ventaja: No necesitas escribir la URL completa. Perfecto para tu propio sitio.

2. 🌐 Enlaces absolutos (sitios externos)

<a href="https://www.google.com">Google</a>
<a href="https://github.com">GitHub</a>
<a href="https://youtube.com">YouTube</a>

⚠️ Importante: Siempre incluye https:// al inicio. Usa target="_blank" para abrir en nueva pestaña.

3. ⚓ Anclas (misma página)

<a href="#inicio">Ir al inicio</a> ← Salta a id="inicio"
<a href="#contacto">Ir a contacto</a> ← Salta a id="contacto"

<section id="contacto"> ← Destino del enlace
  <h2>Contáctanos</h2>
</section>

💡 Uso típico: Tablas de contenido, "volver arriba", menús de una página (landing pages).

4. 📧 Enlaces especiales

<a href="mailto:info@ejemplo.com">Email</a> ← Abre cliente de correo
<a href="tel:+34123456789">Llamar</a> ← Abre marcador en móvil
<a href="sms:+34123456789">SMS</a> ← Abre app de mensajes
<a href="whatsapp://send?text=Hola">WhatsApp</a> ← Abre WhatsApp

📱 Muy útil para: Páginas de contacto, landing pages, sitios de negocios.

UX: Reglas de Oro para Navegación

Una buena navegación puede hacer o romper tu sitio web. Sigue estas reglas:

✅ 1. Máximo 7 elementos en el menú principal

El cerebro humano procesa mejor hasta 7 opciones. Más de eso = confusión.

✅ BIEN: Inicio | Productos | Blog | Acerca | Contacto (5 opciones)
❌ MAL: 15 opciones en el menú = usuario perdido

✅ 2. La opción activa debe destacarse

El usuario debe saber dónde está. Usa colores, subrayados o fondos diferentes.

← Inicio (activa) Blog Contacto

✅ 3. Nombres claros, NO creativos

Los usuarios escanean rápidamente. Usa palabras comunes y predecibles.

✅ BIEN: "Contacto", "Productos", "Acerca de nosotros"
❌ MAL: "¡Escríbenos!", "Nuestras joyas", "Conócenos mejor"

✅ 4. El logo debe llevar al inicio

Es una convención universal. Los usuarios esperan que al hacer clic en el logo vuelvan a la página principal.

<a href="/"><img src="logo.png" alt="Logo"></a>

✅ 5. Navegación consistente en todas las páginas

El menú debe verse IGUAL en todas las páginas. No muevas elementos ni cambies el orden.

✅ 6. Responsive: Menú hamburguesa en móviles

En pantallas pequeñas, oculta el menú detrás del icono ☰ para ahorrar espacio.

Accesibilidad en Navegación

Haz que tu navegación sea usable para TODOS, incluyendo personas con discapacidades:

🎯 Atributos de Accesibilidad

1. aria-label para contexto

<nav aria-label="Navegación principal">
  <!-- Los lectores de pantalla anuncian "Navegación principal" -->
</nav>

<nav aria-label="Migas de pan">
  Inicio › Productos › Laptops
</nav>

2. aria-current para página activa

<a href="/" aria-current="page">Inicio</a> ← Indica página actual

3. Navegación por teclado

Los enlaces deben ser navegables con TAB. El usuario debe ver un outline (borde) al enfocar cada enlace.

⚠️ NUNCA hagas: a:focus { outline: none; }
Esto elimina la accesibilidad por teclado.
